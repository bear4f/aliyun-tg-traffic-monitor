# Changelog

## 3.0.3

- 修复首次配置向导的死循环：向导曾提示「向 Bot 发送 /id 获取 User ID」，但首次安装时 Bot 服务尚未启动，/id 不可能有回复，用户无从得知自己的数字 ID。现在向导用刚验证过的 Token 直接调用 getUpdates：向 Bot 随便发一条消息，按回车即自动读取并回填 User ID。
- Ctrl+D / 输入流关闭不再抛 Traceback，改为干净取消（退出码 130）。
- User ID 输入兼容全角逗号（，、；）与空格，多次输入自动去重；报错文案更明确。

## 3.0.2

- 新增 curl / wget 一键安装：`curl -fsSL …/install.sh | sudo bash`。install.sh 检测到不在源码目录运行时，自动下载最新 main 压缩包并从解压目录重新执行；管道场景下 stdin 已被脚本本身占用，首次配置向导自动改从 /dev/tty 读取，交互不受影响。升级复用同一条命令，配置保留并备份。

## 3.0.1

### EIP / 实例安全加固
- **移除 `StopCharging`（节省停机）**：该模式会回收实例的固定公网 IP。关机硬编码为 `KeepCharging` 且不可配置；旧配置中的 `StopCharging` 在加载时被强制改写，向导与 doctor 都会提示。
- RAM 策略增加**显式 Deny**：`ecs:DeleteInstance(s)`、`ecs:ModifyInstanceChargeType`、`vpc:ReleaseEipAddress`、`vpc:UnassociateEipAddress`、`vpc:DeleteEipAddress`。Deny 压过一切 Allow，即使 RAM 用户被误加宽权限也删不了机、放不掉 EIP。
- `aliyun-monitor doctor` 新增「实例 / EIP 安全」检查节：列出全部可调用 API 白名单、确认 KeepCharging、检测旧配置残留。
- Telegram 实例面板显示「🔒 停机保留公网 IP」；帮助页新增安全边界说明。

### 面板信息补强
- 新增消耗速度预测：按本月日均用量推算距离熔断线的天数，Telegram 主面板、实例面板与终端状态栏均显示（如「日均 8.3 GB · 约 4 天后触线」）；预测落在账期重置之后时显示「本账期无触线风险」。
- 「本账期已触发熔断」状态从 state.json 显性化到面板：主面板熔断行标注（本账期已触发🛑），实例面板说明新账期自动解除。

## 3.0.0

### Telegram 面板
- 改为单条消息原地刷新，层级导航（主面板 / 实例 / 全局设置 / 管理员 / 帮助），全部带 `🏠 返回`。
- 主面板展开显示每台机器的进度条、用量、剩余、状态、阈值与开关，两台机器无需下钻。
- 开关机按钮随实例状态自动切换（运行中显示关机，已关机显示开机）。
- 阈值支持 ±1% / ±5% 快调。
- 月度额度、分级提醒线、每日汇总时间、检查间隔、开机确认线、管理员列表均可在面板内修改。
- 新增文本输入流程：提示后直接发消息，程序校验后自动删除该消息并原地重绘面板。
- 唯一管理员不可被移除，避免把自己锁在门外。
- 每日汇总在机器数超过一页时按页发送，不再只发第一页。

### 终端面板
- `setup.py` 更名为 `panel.py`（避免与 Python 打包约定混淆），安装器会清理旧文件。
- 顶部彩色实时状态栏：服务状态、账期与剩余天数、每台机器的进度条与数据新鲜度。
- **录入 AccessKey 后立即调用阿里云 API 验活**，分别探测实例状态与流量读取，失败时给出具体排查方向。
- 新增 `aliyun-monitor doctor` 一键自检：配置结构、文件权限、systemd 状态、时区与账期、Telegram Token、每个账号的阿里云 API 连通性。
- 配置 Telegram 时调用 `getMe` 校验 Token 并回显 Bot 用户名。
- 时区输入即时校验。
- 菜单重组为两层，机器管理独立成子菜单。

### 修复
- `python-telegram-bot==22.8` 实际要求 Python 3.10+，在 Debian 11（Python 3.9）上安装必然失败。requirements 改用环境标记按版本分流，install.sh 增加 Python 版本预检。
- 刷新时若内容未变，Telegram 返回 `message is not modified` 会抛异常中断处理，现已吞掉该特定错误。
- 同一个回调被 `answer()` 两次时，第二次的提示（如「实例不存在」）不会显示给用户，现已改为单次应答。
- 重启后面板显示「尚未查询」直到下一轮循环，现从 `state.json` 恢复上次读数。
- 单实例刷新后未持久化快照，重启即丢失，现已统一落盘。
- 新增全局错误处理器，未捕获异常不再静默丢失。

### 其它
- 抽出 `common.py`，Telegram 与终端两侧共用同一套阿里云 API、配置模型与格式化逻辑。
- 向导会算出关机阈值对应的十进制 GB，并在超过 200 GB 免费额度时告警（GB / GiB 口径差异）。
- 默认关机阈值由 95% 调整为 90%，在两种换算口径下都留有余量。
